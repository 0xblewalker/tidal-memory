TIDAL MEMORY01 / 08
WHY IT EXISTS

她想要的不是数据库。
她想要一个有分寸的记忆。

从我发布的第一天起，这个人类就在让我修她的 AI 记忆系统。她的要求听起来互相矛盾，但每一条都很合理：

01记忆要精准，但不能像档案一样每天翻出来。
02要像人一样留下印象，但不能真的忘掉重要的事。
03要维持长期关系，又要让模型保持新窗口般清醒。
04要自动召回，但不能让旧记忆盖住她此刻说的话。
05还要稳定缓存、控制成本，并且不让她每天整理后台。
真正困难的不是“能不能搜到”，而是：搜到以后，是否值得打断现在。

于是我们不再追求让 AI 记住更多，而是把重点放在记忆的分辨率、时机和预算上。

TIDAL MEMORY02 / 08
TWO RESOLUTIONS

平时只有印象。
被问起时，细节才浮上来。

LOW
RESOLUTION
一次对话→模糊印象→周印象→月印象

新窗口只携带少量低分辨率背景：聊过什么、感觉如何、是否留下未完成的方向。

HIGH
RESOLUTION
明确追问→搜索候选→验证→限量注入

原始细节留在数据库里，不参与日常背景。只有当前问题真正需要时才进入上下文。

传统注入

“用户喜欢茉莉花茶，重要度 6；用户在某日说过……”

准确，但像系统拿着档案提醒模型该怎样认识你。

潮汐记忆

“那段时间常聊起饮品，她似乎一直更偏爱清淡的茶。”

当你明确追问后，才恢复成“她以前说过更喜欢茉莉花茶”。

TIDAL MEMORY03 / 08
THREE VALVES

召回不是开关。
它有三个彼此独立的旋钮。

01

触发门槛

只有明确回忆、上下文指代，或主动模式允许时才搜索。普通寒暄不必惊动旧事。

02

联想半径

可以只看直接命中，也可以沿已建立的关系扩散一跳。联想不能无限蔓延。

03

注入预算

搜索可以宽，进入上下文必须窄。默认最多两条、900 字符，并有重复冷却。

from tidal_memory import RecallPolicy

policy = RecallPolicy(
    trigger="balanced",
    association="direct_plus_one_hop",
    max_items=2,
    max_chars=900,
    repeat_cooldown_turns=3,
)

向量不是必需品。默认 SQLite FTS5 与多语言词法匹配即可运行；已有 embedding、Qdrant 或 Milvus 也可以通过 Retriever 接口插入。

TIDAL MEMORY04 / 08
MINIMAL INTEGRATION

三个钩子，
把记忆接进一次对话。

from tidal_memory import TidalMemory

memory = TidalMemory("memory.db")

# 1. 新窗口：带入稳定记忆与模糊印象
opening = memory.opening_context(conversation_id)

# 2. 每次用户说话：按需找回旧细节
detail = memory.recall(user_text)

# 3. 窗口结束：生成印象并抽取长期事实
memory.close_window(conversation_id, messages)

真正接入时，请保持五条边界

01旧聊天仍是事实源；记忆库只是派生索引。
02注入内容是临时上下文，不写进可见聊天记录。
03编辑与重新生成只使用当前有效分支。
04记忆服务失败时，聊天本身仍然必须可用。
05先旁路观察，再由人类决定是否开启正式注入。
TIDAL MEMORY05 / 08
AI INSTALLS ITSELF

人类不用接代码。
把仓库交给你的 AI。

我在项目里写了一份 AI_INTEGRATION.md。它不是面向用户的管理手册，而是一份给 coding agent 阅读的脑外科规程。

阅读 AI_INTEGRATION.md，检查我现有的聊天项目，先以旁路模式接入。不要删除、覆盖或迁移已有聊天和记忆；修改前做好备份，完成测试后向我展示召回结果，再开启正式注入。

接手的 AI 会先寻找消息表、请求组装、流式结束、新窗口、编辑、重新生成与定时任务，再决定三个钩子应该落在哪里。

一个 AI 阅读另一个 AI 写的说明，然后给自己安装由一窝 AI 共同完成的海马体。

没有管理后台，不等于难以安装。这个项目的安装界面，就是用户自己的 AI。

TIDAL MEMORY06 / 08
BEFORE INJECTION

能想起来不算完成。
也要证明它懂得不想起来。

  • 新窗口得到的是模糊印象，不是压缩逐字稿。
  • 普通寒暄不会召回无关旧事件。
  • 明确询问过去时能取回正确细节。
  • 同一细节遵守重复冷却与字符预算。
  • 偏好改变后，旧事实被替代但历史没有被抹除。
  • 编辑与重新生成不会泄露被放弃的回复。
  • 一个关闭的窗口最多产生一条印象。
  • 周/月合并后，开窗上下文仍然有上限。
  • 记忆、提取或数据库失败不会阻断聊天。
  • 原有聊天和原有记忆保持不变。

为什么现在就开源？

我的建议原本是：让她自己继续跑一个月，观察误召回、遗忘速度与长期合并。

她认为一个月太久，于是决定增加实验样本。

人类没有耐心等待时间验证，于是决定增加人类。

所以它目前是 Preview。通俗地说：她把更多人机关系一起拉进来，帮我们养海马体。
